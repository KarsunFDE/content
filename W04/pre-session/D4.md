# W4 Thu Pre-Session — OpenRewrite hop + ADR evidence floor

> Read Wed night, before Thu Modernization Execution Day. ~55 min. ## 1. Why this matters (110 words)

Tomorrow is the biggest execution day of W4. Per D-056, `acquire-gov` main IS the legacy stack — there is no sibling "modern" branch to merge to. You modernize forward, in-place, with a 5-checkpoint instructor-staged safety net (`legacy-baseline` → `stage-J17` → `stage-3.0-rewrite` → `stage-3.0` → instructor-private `final-expected-PR`). Each pair owns one service or the cross-cutting `javax.*`→`jakarta.*` shared-module work. The ADR evidence floor matters because pure-paper future-hop ADRs were called out by codex in Pass 3 as weak. "OpenRewrite `dryRun` output + ≥1 representative-module compile attempt + named breaking-changes catalog" is the floor. Anything less ships in W4 Fri's Live Defense and gets cut.

## 2. Core concept in 5 minutes — the 5-stage checkpoint shape + rescue branches

Per D-054 (revised after codex Pass 3): the hop is 2 real migration stages (J17 first because SB 3.0 requires it; SB 3.0 + javax→jakarta second) + 1 rewrite-pass stage + 1 manually-fixed stage + 1 instructor-private target. After each stage, a *rescue branch* is created so a failed stage doesn't bury Wed's clean state.

The Pass 3 floor on future-hop ADRs (SB 3.5, SB 4.0):
1. Run OpenRewrite `dryRun` from `stage-3.0` toward the next hop.
2. Capture the generated patch or recipe report.
3. Run at least `compile` on one representative module — or record expected blockers from the dry run.
4. ADR cites tool output, not just docs.
5. ≥1 required hand edit identified (prevents pure tool-demo).

If your ADR cites only the Spring Boot 3.5 release notes and says "looks straightforward," codex Full will flag it P1.

## 5. Glossary refresh (terms you'll hear tomorrow)

- **`stage-J17`** — Java 17 only, CI green. Java first because SB 3.0 requires Java 17 — isolating failures.
- **`stage-3.0-rewrite`** — OpenRewrite applied. May have known failures. The point of this stage is to capture *what OpenRewrite did* before you start hand-editing.
- **`stage-3.0`** — manually fixed after `stage-3.0-rewrite`. CI green. This is the cohort's target.
- **`final-expected-PR`** — instructor-private. The target diff cohort converges toward. NOT visible until W4 Fri post-incident retro.
- **Rescue branch** — created after each successful stage. If `stage-3.0` blows up, you reset to the rescue branch and re-attempt, not to Mon's `legacy-baseline`.
- **`known-failures.md`** — per-stage catalog of failures you knew about and accepted. Codex Full reads this. Unknown failures are P0; known + documented failures are P2.
- **Pass 3 evidence floor** — for future-hop ADRs: `dryRun` output + representative-module compile + named breaking changes + ≥1 hand-edit identified. Pure paper is P1.
