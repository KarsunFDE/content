## 2. Core concept in 5 minutes — OpenRewrite for brownfield modernization

OpenRewrite is a refactoring engine that applies *recipes* (declarative AST transforms) to a codebase. For Spring Boot 2.7→3.0 + Java 11→17 + javax→jakarta, OpenRewrite ships the `org.openrewrite.java.spring.boot3.UpgradeSpringBoot_3_0` recipe — one command, hundreds of edits, deterministic output. It doesn't do everything (Pass 3 of D-054 floor: *"≥1 required hand edit after OpenRewrite to prevent pure tool-demo"*), but it gets the floor right.

The Thu hop has 5 instructor-staged checkpoints: `legacy-baseline` (start) → `stage-J17` → `stage-3.0-rewrite` (OpenRewrite applied, may have known failures) → `stage-3.0` (manually fixed, CI green) → `final-expected-PR` (instructor-private). You'll work the J17 + 3.0-rewrite + 3.0 stages.

## 5. Glossary refresh (terms you'll hear tomorrow)

- **OpenRewrite recipe** — declarative AST transform that applies to a Java codebase deterministically. Recipes compose; the SB 3.0 upgrade is a composite of ~30 sub-recipes.
- **`dryRun`** — OpenRewrite mode that emits a patch without applying it. The D-054 Pass 3 evidence-floor for future-hop ADRs requires `dryRun` output, not pure docs.
- **`javax.*` → `jakarta.*`** — Jakarta EE 9 namespace move. Spring Boot 3.0 baselines on this. Every `javax.servlet`, `javax.persistence`, `javax.validation` import flips.
- **Debt item 12 (lint disabled)** — `.github/workflows/ci.yml` line invariant: `if: false` on the lint job. Tomorrow you find this; the OIG-Findings-Tracker meta-joke uses this as evidence finding #1.
- **Debt item 8 (frontend hardcode)** — `UI/src/app/dashboard/solicitation-list.component.ts` (or equivalent) hits `http://localhost:8081/api/solicitations` directly. Bypasses `api-gateway`. Smells when you trace a request and find no gateway log.
- **Plan-day vs. living-spec** — Plan Day is when you commit; living-spec is what the spec actually *does* on Tuesday morning when it meets the codebase.
