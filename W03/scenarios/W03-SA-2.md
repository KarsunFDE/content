# W03-SA-2 — LangGraph vs LangChain agents vs hand-rolled Python coordinator

## 1. The client constraint

> "Your evaluator → consensus → SSA flow has HITL gates at SSA approval (FAR 15.308) and award publication (FAR 5.705). Both are *hard* gates — the graph blocks indefinitely until human input arrives. The SSA may not approve the SSDD draft for 18 hours. Your framework choice has to support that. It also has to integrate with our existing ai-orchestrator FastAPI service + Pydantic schemas + Bedrock InvokeModel direct (we are NOT using Agents-for-Bedrock this week per D-060). And — important — the cohort will hand this off to a future engineering team. The framework you pick is the framework they inherit. Defend the call."
>
> — Karsun delivery lead, captured during the W3 Wed scenario-research slot

## 2. The default programme answer

**LangGraph** (state machine with `interrupt_before` for HITL + `PostgresSaver` for checkpointing). LangGraph is the v1.0+ LangChain-stack primary primitive for agent state machines; it has first-class support for hard interrupts, checkpointing, time-travel/replay, and LangSmith tracing. The framework is what makes the 18-hour SSA-approval-gap viable. We teach the pattern programme-wide.

## 3. Candidate alternatives the pair evaluates

| Alternative | One-line capability | Maturity |
|-------------|---------------------|----------|
| LangChain agents (`langchain.agents`, `create_react_agent` outside LangGraph) | Pre-LangGraph LangChain agent abstraction; works but lacks state machine modeling | Deprecated direction in LangChain v1.0 (verified 2026-05-23 via /web-research); `Chain` class removed; LCEL `|` pipe no longer central — see `skills/tech-research/references/known-bad-patterns.yml` |
| Hand-rolled Python coordinator | Plain Python `async def` + explicit if/else + custom checkpointing-to-Postgres + custom HITL pause-resume | Always-available; you wrote it; every defect is yours |

## 4. Comparison dimensions the pair scores (1–5)

For each alternative (and the default), score on:

- **Capability fit** — can the framework express hard `interrupt_before` with PostgresSaver-equivalent persistence? Can it survive a FastAPI restart while the SSA is away? Can it audit the resume against FAR 15.308?
- **Cost shape** — runtime cost is roughly equal (Bedrock dominates). Maintenance cost is the variable: LangGraph upgrades quarterly; LangChain agents are deprecated; hand-rolled has zero framework cost + 100% maintenance cost. Project across 24 months.
- **Operational burden** — debugging: LangGraph + LangSmith integration vs LangChain agents + ad-hoc tracing vs hand-rolled + print-statements. What does the 23:00 page look like for each?
- **Security posture** — multi-tenant boundary (Item 10) + audit completeness (Item 2 + Item 6). Where does each framework give you guard-rails? Where does each require you to remember the right thing?
- **Maturity** — LangGraph v0.6 series stable, weekly minor releases (verified 2026-05-23 via /web-research); LangChain agents deprecated trajectory; hand-rolled is whatever you build.
- **Exit cost** — switching framework in 24 months. LangGraph → hand-rolled (decompose the state machine into your own code — feasible). LangGraph → LangChain agents (downgrade — not advised). Hand-rolled → anything (start over).

## 8. Anti-patterns to call out in defense

- **Citing LangChain v0.x examples.** If your reasoning leans on `chain.run()` or `LLMChain` or `RetrievalQA.from_chain_type`, you're defending a deprecated path. Codex at Near-full strictness flags this. (Per `known-bad-patterns.yml`.)
- **Treating LangGraph and LangChain agents as interchangeable.** They're not. LangGraph is the v1.0+ direction. LangChain agents (in `langchain.agents`) are the deprecated direction. The pair must know this distinction cold.
- **Hand-rolled "for performance."** Performance is not the variable. Bedrock latency dominates by 100×. Framework overhead is noise.
- **"We picked LangGraph because it's what the programme teaches."** That's loyalty, not engineering. The defense has to be that LangGraph *is the right call for the constraints*, not that it's what was taught.
- **Forgetting the 24-month maintenance cost.** Hand-rolled looks cheap on Day 1. Day 731 it's a full-time job.
- **Skipping the LangSmith integration argument.** LangGraph + LangSmith is a *single integration*. Hand-rolled = build your own tracing. LangChain agents = ad-hoc + deprecated. This is a load-bearing distinction.
