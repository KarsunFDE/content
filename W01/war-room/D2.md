## Act 1 — Claude Code orientation (25 min)

Before touching the codebase, the cohort sees what Claude Code is and what makes it different from chat-window LLM use.

**Demonstrated patterns:**

1. **Install + auth verification** — each learner runs `claude --version` and confirms an authenticated session. Instructor walks any auth failures live. *No `claude doctor` opt-outs — fix the breakage now.*
2. **Plugin / MCP intro** — instructor opens `.mcp.json` at the `acquire-gov` repo root, walks the Atlassian Rovo MCP server config + how MCP tools surface inside Claude Code (`mcp__atlassian-rovo__*`). Names the difference between an MCP tool, a built-in tool, and a custom skill.
3. **Custom-skill tour** — instructor walks `.claude/skills/` in the repo. Picks 2–3 skills (`lms-feature` orchestrator + `lms-debug` + one PII skill) and names: *what's a skill? where does Claude Code discover it? when does it auto-invoke vs require explicit invocation?* Cohort sees the skill-discovery walk-up pattern.
4. **Statusline + hooks (30 sec each)** — show `.claude/statusline/` + `.claude/hooks/` exist; flag that hooks block sensitive reads/writes (`.env`, secrets). Don't deep-dive; cohort will encounter these live across the week.

**Cohort prompts (instructor invites questions):**
- *"When does Claude Code use an MCP tool vs a custom skill vs the model alone?"*
- *"What does it cost me, latency-wise, to invoke a skill vs ask Claude directly?"*

## Act 2 — Repo first-look (30 min)

Instructor opens the `acquire-gov` repo. Shows nothing slide-style — only the repo and Claude Code.

**Demonstrated patterns:**

1. *"Claude, walk me through what this repo does. Start at the README, then tell me what each top-level directory contains."* — narrate Claude's output, point out useful sections, ignore the rest.
2. *"Claude, this looks like a microservices app. Diagram the service-to-service call graph for me."* — Claude produces an ASCII diagram or Mermaid; instructor compares against `training-project/README.md` architecture section.
3. *"Claude, what would I run first to see this thing alive?"* — Claude finds the docker-compose file; instructor runs `docker-compose up` together with the cohort.

**Cohort prompts:**
- *"What's the difference between asking Claude to explain a file vs explain a flow?"*
- *"How do I know when Claude is making something up?"* (introduces hallucination check: verify against the repo's actual files).
- *"What's the right prompt when I have no idea what the code does?"*

## Act 3 — Angular SPA tour + named brownfield gaps (35 min)

Instructor walks the `frontend/` Angular 17+ SPA — routing, components, services — and names three specific brownfield-debt items live (cohort doesn't fix; surface only).

**Tour beats:**

1. **Routing + lazy-loaded modules.** Walk `frontend/src/app/app.routes.ts`. *"What does this routing tell you about user roles?"* — surfaces tenant-shape assumptions.
2. **Components vs services vs guards** — quick refresher pattern, framed for the Java-leaning learners.
3. **Three brownfield gaps named live** (these are 3 of the 12 deliberate debt items per `training-project/README.md`):
   - **Hardcoded service URL bypassing the gateway** — `frontend/src/app/services/solicitation.service.ts` calls `solicitation-service:8081` directly instead of going through `api-gateway:8080`. Why this matters: breaks tenant routing, breaks JWT validation point, breaks rate limiting. *"Where else might this pattern hide?"*
   - **Missing form validation** — `frontend/src/app/solicitation-new/` posts to the backend with no client-side validation. Cohort can submit empty solicitations end-to-end. *"What does this cost in real Bedrock spend if a user mashes the button?"*
   - **Dead routes** — `app.routes.ts` has 2 routes (`/reports`, `/admin/audit`) wired up to components that no longer exist (compiled errors hidden by `--skip-tests` in CI). *"How did this ship?"* — opens the dev-velocity-vs-quality conversation.

Cohort doesn't fix any of this today. Surfacing is the curriculum; fixing is W4 (modernization week).
