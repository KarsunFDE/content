### Block 1 (55 min) — Microservices Foundation walkthrough + containerization deep-dive

Instructor-led walkthrough of `acquire-gov` architecture against the running `docker-compose` stack. Cohort follows along with the repo open; brownfield-debt items annotated live (added to `weeks/W01/brownfield-debt.md` in Block 2).

**Architecture pass (25 min):**
- Service mesh: Angular SPA → API Gateway → Spring Boot services (`solicitation-service`, `evaluation-service`) → Python/FastAPI `ai-orchestrator`.
- Service boundaries + bounded contexts — why these splits, where they'd bend under federal-scale load.
- OAuth2 + JWT auth flow end-to-end — token mint → gateway validation → service-to-service propagation. **Deliberate gap:** one service skips claim re-validation (debt item 1) — flag, don't fix here.
- Structured logging + correlation IDs — intentionally inconsistent format across services (debt item) so cohort sees the AIOps problem before W5's solution.
- Distributed-systems basics — synchronous vs async paths in this codebase, where retries live, where they don't.

**Containerization pass (30 min):**
- Multi-stage Dockerfiles — read `ai-orchestrator/Dockerfile`; note `python:3.11-slim` pin (not `:latest`) as the *right* call.
- **`:latest`-tag antipattern** — debt item 11 lives in `services/solicitation-service/Dockerfile`. Why it's broken: non-reproducible builds, surprise updates, no rollback target.
- **Missing healthchecks** — debt item 7. Compose has no `healthcheck:` on Postgres or the AI orchestrator. Why it matters: dependent services start before deps are ready; cascading 5xx on cold-up.
- **Postgres volume persistence gap** — also debt item 7. Volume mount is `tmpfs`-equivalent in dev config; data dies on `docker-compose down -v`. **Sub-deep-dive defers to Thu AM first 30 min per PLAN.md L61** if Block 1 runs long.
- Compose networking + service-discovery DNS — how `solicitation-service` finds `ai-orchestrator` by Compose service name, why that breaks in production without a real service registry.

## Anti-patterns to avoid Wed

- **Letting the Galent demo run over.** Hard stop at 12:00; afternoon needs full 3 hours.
- **Picking the comparative ADR "winner" for the cohort.** The exercise is the comparison, not the conclusion.
- **Letting pairs scaffold a Pair Project repo for *the same domain* as the training-project.** Pair Projects need to be different domains so the cohort experiences AI adoption in a context they're not already steeped in.
- **Skipping the public-material framing for ReDuX.** Cohort needs to internalize "we don't have access to ReDuX internals" — same posture they'll have at most federal engagements where client systems are proprietary.
- **Fixing brownfield debt items during the walkthrough.** Block 1 is *inventory*, not *modernization*. Items 7 + 11 surface in the walkthrough; pair logs them in `brownfield-debt.md` and moves on. Fix vector decisions land W4 (modernization phase).
- **Over-running Block 1 into Block 4's repo-init budget.** Block 1's containerization pass has a 30-min cap; if Postgres-volume-persistence isn't done by 13:55, defer the deep-dive to Thu AM per PLAN.md L61 — do not steal time from repo init, which has a hard W1 Wed EOD deliverable.
