# W1 Wed Pre-Session — LLM Engineering Essentials primer

> Read **after Wed's Galent demo** and **before Thu morning**. ~45 min. Wed is the Galent slot — this pre-session frames Thu, when the cohort's first production LLM PR lands.

## 1. LLMs as engineering systems — not magic (10 min)

By Thu the cohort starts treating Bedrock invocations as production calls — not "fancy text". Production AI engineering requires the same discipline as any other API integration: structured I/O, validation, retry, observability, cost tracking, security.

The mental model:

- **Input is a structured prompt**, not free text. Persona + constraints + grounding + output format are all engineering decisions, not creative writing.
- **Output is a typed contract**, not a string. Pydantic on Python side. Bean Validation on Spring side. Contract tests across both.
- **Failure modes are real**: hallucination, stale knowledge, truncation, JSON corruption, rate limits, cost overruns.
- **HITL is a production pattern**, not a UX afterthought.

## 2. Hallucination failure modes (10 min)

Hallucination = the model produces plausible-but-wrong output. In federal acquisitions, this kills the project (and possibly the contract). Categories you'll see Thu:

- **Confabulation** — model invents a citation, statute, or precedent that doesn't exist.
- **Stale knowledge** — model's training data is older than the FAR/DFARS amendment that just dropped.
- **Over-confidence in plausible outputs** — model returns a paragraph that *sounds* right but contradicts the source corpus.
- **Format drift** — model returns prose when you asked for JSON, or vice versa.
- **Truncation** — model hits context window mid-sentence and stops.

Mitigation lives in:
- **Structured output validation** (Pydantic strict mode) — fail fast on format drift.
- **Grounding via RAG** (W2) — the model only cites what's in the retrieved chunks.
- **HITL gates** (W3 + W4) — for irreversible outputs, a human approves before send.
- **Eval harness** (W2 Fri) — regression suite catches drift over time.

## 3. Model selection criteria for federal contexts (5 min)

The programme anchor is AWS Bedrock with Claude. Read `training-project/docs/adrs/0002-bedrock-as-llm-anchor.md` for the rationale.

When you're asked to defend the choice (W1 Fri's "Bedrock vs OpenAI direct" scenario-alternatives prompt is the first time):

- **FedRAMP authorization** — Bedrock has FedRAMP High in GovCloud regions. OpenAI direct doesn't, as of Q1 2026.
- **Data residency** — Bedrock keeps inference data in-region. OpenAI direct has different controls.
- **GovCloud availability** — Bedrock GovCloud is the default for federal workloads. OpenAI has no GovCloud equivalent.
- **Cross-region inference** — Bedrock's CRIS feature gives higher throughput without leaving the FedRAMP boundary.

When OpenAI direct *does* win:
- Commercial-only workloads with no federal data.
- Model capabilities Bedrock-Claude doesn't yet have (uncommon in 2026).

## 4. AWS Bedrock model invocation — what you'll do Thu (15 min)

Thursday's first PR lands in `services/ai-orchestrator/`. The starting state (read `app/main.py`):

```python
@app.post("/draft-solicitation")
def draft_solicitation(req: DraftRequest) -> dict[str, Any]:
    # DELIBERATE GAPS: no system prompt, no streaming, no retry, no cost tracking,
    #                  no structured-output schema. Cohort fixes in W1 Thu–Fri.
    prompt = f"Draft a federal solicitation paragraph about: {req.topic}..."
    body = json.dumps({"anthropic_version": "bedrock-2023-05-31", ...})
    response = bedrock_runtime.invoke_model(modelId=BEDROCK_MODEL_ID, body=body, ...)
    raw = json.loads(response["body"].read())
    return raw   # ← raw Bedrock response, downstream Spring hits KeyError
```

Thursday's PR adds:

- **System prompt architecture** — persona (contracting officer drafting assistant), constraints (FAR/DFARS-compliant language), output format (JSON with named fields), grounding (RAG comes W2; W1 uses naive system prompt only).
- **Streaming** — SSE response so the Angular UI doesn't hang for 30 seconds.
- **Retry logic** — exponential backoff + jitter on 429s and 5xxs; circuit-break on persistent failure.
- **Cost tracking** — log per-request input/output tokens; tag by feature for per-tenant budget tracking (W5 Tue AIOps).
