## 1. What is the FDE Intensive? (8 min)

The Forward Deployed Engineer (FDE) is a Revature programme that prepares software engineers to lead AI-enabled engagements at federal-modernization clients. The 6-week intensive (Karsun-FDE v2) is **Phase 2** of the broader FDE arc — Phase 1 is foundational learning (pre-cohort); Phase 3 is 6 months of on-the-job development after the cohort ends.

**This intensive is condensed.** Each day carries up to 12 topics; mornings are war-room incidents in a real codebase, early afternoons are pair-led practical work, late afternoons are conceptual previews of tomorrow. The format mirrors how an FDE actually works at a client — drop into a real system, comprehend it under pressure, ship work that holds up.

The cohort runs against a real codebase — **`acquire-gov`** — a federal-acquisitions training project deliberately seeded with brownfield debt. The cohort modernises it across the 6 weeks while building a separate continuous **Pair Project** that lives in each pair's own repo.

## 2. The 12 FDE situations (12 min)

The FDE programme is structured around 12 *situations* an FDE encounters at engagement. Every day touches at least one. Read these now; you'll see them named throughout the next 6 weeks:

1. **Data comprehension** — what is this data, who owns it, what's stale, what's missing?
2. **Tech stack comprehension** — what is this codebase, why is it shaped this way, where are the seams?
3. **Requirements synthesis** — what does the client actually need (vs what they asked for)?
4. **Estimation** — what's this going to cost in time, money, complexity?
5. **6R and system mapping** — Retain / Rehost / Replatform / Refactor / Repurchase / Retire. Knowledge-graph + Context-graph thinking (W3).
6. **Non-functional requirements** — performance, security, observability, compliance, accessibility.
7. **Cloud thinking** — what does production look like on AWS (or another provider)?
8. **DevOps thinking** — how does code get from a PR into production safely?
9. **Database thinking** — relational, document, vector — which when?
10. **Security thinking** — OWASP, OWASP LLM Top 10, multi-tenant boundaries, secret management.
11. **Architecture thinking** — ADRs, trade-off framing, defending decisions under probing.
12. **Design thinking** — user outcomes, not feature lists; what does the contracting officer actually need?

## 5. Tools we'll get you set up with (3 min)

**Container-first.** `acquire-gov` ships as a `docker-compose` stack — every service runs inside a container with its own pinned runtime (Java 11 on the legacy baseline → Java 17 after the W4 modernization hop; Python 3.11; Node ~20). You don't need to install those runtimes on your host. `docker-compose up` is the truth source.

**Required on your host:**

- **Docker Desktop** (latest stable) — the one hard prerequisite. Everything else runs inside containers.
- **Git** — clone the repos.
- **VS Code or your editor of choice** — for reading + editing code.

**Provisioned during the session:**

- **AWS CLI v2** + `aws configure sso` for the cohort SSO link (we'll walk you through this Tuesday).
- **Claude Code** — accounts may not be live for Tuesday; if they're not, you'll read the repo as-is and we'll bring Claude Code online as soon as accounts land (Wed at latest). Install path: `https://docs.claude.com/en/docs/claude-code`. The Claude Code extension for VS Code goes on after your account is live.

**Optional — for IDE convenience only:**

If your editor wants a JDK / Python / Node available on host for autocomplete, navigation, and debugging:

- **JDK 21 LTS** (via [SDKMAN](https://sdkman.io/): `sdk install java 21-tem`) — handles Java 11/17/21 codebases via `--release` flag; you don't need to install Java 11 separately. SDKMAN lets you `sdk use java 17-tem` later if you want to match the W4 modernization target exactly.
- **Python 3.11+** (via [pyenv](https://github.com/pyenv/pyenv) or your package manager) — only if you're going to run pytest/ruff outside the container.
- **Node 20 LTS** (via [nvm](https://github.com/nvm-sh/nvm)) — only if you're going to run `ng serve` outside the container.

**If you arrived with only Docker Desktop installed, you can do the entire programme.** The containers handle every runtime version through every modernization hop. Host installs are convenience, not requirement. We'll work through setup together Tuesday — bring the laptop you'll actually use, and we'll go from there.
